# STT 与 TTS 服务选型建议

> 目标场景：西溪湿地摊位、中文游客、户外噪声、每轮 3—15 秒语音、算命先生需要沉稳而不阴森的声音。

## 一、先给结论

### 最省开发：OpenAI Audio

适合当前应用最快接入。现有服务端已经按 OpenAI 风格实现：

- STT：`/audio/transcriptions`
- TTS：`/audio/speech`

优点：只需填写环境变量，改动最少；文本模型、STT、TTS 可以共用一套账号体系。

声音建议：选择偏低沉的男声，并把语速设为约 `0.88—0.94`。如果接口支持自然语言声音指令，可使用：

> 一位五十岁左右的中国男性民俗讲述者，声音低沉温和，吐字清楚，语速稍慢，有一点江湖说书人的从容，但不要阴森、夸张或故弄玄虚。

### 中文现场效果优先：阿里云百炼语音或火山引擎语音技术

两者在中文识别、中文音色、国内网络和声音定制上更适合正式摆摊。若要做“有明确人设的算命先生”，优先试听它们的声音复刻或声音设计能力。

代价：接口格式通常不是 OpenAI Audio 完全兼容，需要在 `server/providers.mjs` 增加适配器。

### 最容易做出角色感：ElevenLabs

同时提供 STT 和 TTS，声音设计、克隆和风格控制较强。适合做有辨识度的“先生”音色。

代价：境内现场网络稳定性、价格和中文专有名词发音需要提前实测，并准备国内网络下的降级方案。

## 二、候选服务对比

| 服务 | STT | TTS | 中文与角色音色 | 接入当前应用 | 现场建议 |
|---|---|---|---|---|---|
| OpenAI Audio | 有 | 有 | 中文自然，声音指令灵活 | 最简单，现有接口基本兼容 | 快速上线首选 |
| Microsoft Azure AI Speech | 有 | 有 | 中文神经音色丰富，可细调语速、音高、风格 | 需写 Azure 适配器 | 企业稳定性和精细控制首选 |
| 阿里云百炼/智能语音 | 有 | 有 | 中文识别与 CosyVoice 类音色适配好 | 需写 DashScope/百炼适配器 | 国内正式现场优先候选 |
| 火山引擎语音技术 | 有 | 有 | 中文音色与声音复刻能力强 | 需写火山接口适配器 | 人设音色和国内网络候选 |
| 腾讯云智能语音 | 有 | 有 | 普通话识别稳定，国内服务成熟 | 需写腾讯签名和接口适配器 | 稳定实用、角色感需试听 |
| ElevenLabs | 有 | 有 | 声音设计和克隆突出，中文需实测 | 需写 ElevenLabs 适配器 | 角色感优先、需网络备份 |

## 四、适合算命先生的声音标准

不要只试听“声音好不好听”，应按以下标准打分：

| 维度 | 推荐标准 |
|---|---|
| 年龄感 | 约 45—60 岁，可信但不衰老 |
| 音高 | 中低音，不要刻意压成恐怖低音 |
| 语速 | 普通中文语速的 0.88—0.94 倍 |
| 情绪 | 温和、从容、带一点说书感 |
| 咬字 | “鲁智深、李逵、张顺、安道全、西溪”等必须清楚 |
| 停顿 | 每句 12—25 字后自然停顿 |
| 环境适配 | 小音箱播放后仍能听清辅音和数字 |
| 禁止方向 | 阴森、气声过重、过度神秘、播音腔太强、方言难懂 |

## 五、推荐的试听文案

所有供应商用同一组文案测试，才能公平比较：

### 开场

> 客官请坐。今日想为自己、孩子、父母，还是全家问一份好彩头？

### 对话内数字小术

> 三位数倒过来，大数减小数，再把结果倒过来相加。一百单八将，前来点福。

这段只用于被 Agent 随机选中 1089 小术且游客愿意参与的会话，不应做成独立功能入口。

### 商品推荐

> 先生为你点定：安道全，安到全家成品香囊。你为父母求一份平安，又想直接带走，这件最合适。

### 儿童场景

> 小客官今天点到李逵，快乐加倍。建议和家里人亲手做一只香囊，把共同完成的快乐带回家。

### 头巾场景

> 客官喜欢边游边拍，今日便点张顺的顺风顺水。建议戴一条张顺主题水浒头巾，既应这份好彩头，也方便留下西溪游玩的纪念。

## 六、实际采购前必须测试

1. 西溪现场或相似户外环境的识别准确率。
2. 4G/5G 热点下首包延迟。
3. “李逵”“安道全”等专名的识别和发音。
4. 儿童高音和中老年较轻声音的识别率。
5. API 是否支持流式 STT/TTS。
6. 音频是否允许商业现场播放。
7. 声音克隆是否需要被克隆者授权。
8. 数据是否会用于模型训练以及音频保存期限。
9. 国内访问稳定性、并发限制和价格。
10. 服务失败时能否快速切回浏览器语音或快捷按钮。

## 七、最终推荐顺序

1. **先用 OpenAI 风格语音接口跑通**：开发量最小，验证整个语音闭环。
2. **正式摆摊前同时试听阿里云和火山引擎**：重点比较中文专名、户外识别和男声音色。
3. **如果需要强角色音色，再试听 ElevenLabs**：但必须做国内网络实测。
4. **需要企业级稳定性与 SSML 精细控制时考虑 Azure**。

不建议一开始同时接五家。先选一家完成闭环，再保留浏览器语音作为降级方案。
