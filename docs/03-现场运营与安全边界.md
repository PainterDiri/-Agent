# 现场运营与安全边界

## 一、对外定位

统一称为：

- 民俗娱乐体验。
- 数学小魔术。
- 水浒人物祝福推荐。
- 今日福物点选。

不要称为：

- 真正预测命运。
- 风水大师开运。
- 医疗或健康诊断。
- 保证发财、升学或治病的服务。

界面右上角已经常驻显示“民俗娱乐体验”。

## 二、工作人员开场

> “可以和四福先生说几句话，先生有时会随口带一个数字、颜色或方位小互动。最后它只为您推荐一件水浒福物，不是算真实命运。”

## 三、游客拒绝购买

当游客说“不买”“只玩玩”“没兴趣”时：

1. 立即停止商品引导。
2. 给一句普通祝福。
3. 点击“重新开始”，清空上一位游客会话。

不要用免费体验制造亏欠感。

## 四、儿童使用

- 儿童由监护人陪同。
- 不询问学校、住址、电话号码和真实姓名。
- 不让儿童为了抽隐藏款连续购买福袋。
- 不引导儿童讨论家庭经济、疾病或父母关系。
- 福袋内小零件按实际年龄提示管理。

## 五、敏感话题处理

### 疾病和健康

只说“祝愿平安、身心舒展”。不能诊断、治疗或推荐香囊代替医疗。

### 财务困难

不推荐财运产品，不暗示消费能改变经济状况。可以结束为普通祝福。

### 考试与升学

可以说“祝学习顺利、从容发挥”，不能说“必过”“保上岸”。

### 丧失、灾害与严重情绪危机

停止娱乐式算命和销售。建议游客寻求家人、现场工作人员或专业帮助。

## 六、隐私

- 默认不保存游客音频和聊天内容。
- 浏览器刷新或按“重新开始”后清空当前前端会话。
- 服务端没有数据库。
- 不进行人脸识别。
- 不要求登录、手机号或扫码注册。
- 若未来增加数据统计，只记录匿名数量，不记录原始语音和完整对话。

## 七、密钥安全

- API key 只放在 `.env`。
- `.env` 已加入 `.gitignore`。
- 不把 key 写进 `src/`、HTML 或二维码。
- 不在游客可见屏幕打开 `.env`。
- 活动结束后如使用临时 key，建议立即轮换或撤销。

## 八、网络与排队

- 每组语音互动控制在 2—4 分钟。
- 排队超过三组时，优先让游客使用快捷回答。
- 模型延迟过高时切换 `MOCK_MODE=true`。
- 每次服务完成后点击“重新开始”。
- 工作人员负责拿出推荐商品，不让游客翻动全部库存寻找。

## 九、对话内小术说明

小术没有独立入口，由 Agent 每次会话随机决定是否使用以及使用哪一种。同一会话最多一种，游客不想参与时立即回到普通聊天。

- 西溪景物：水、桥、芦苇分别联想顺势、连接与韧性。
- 方位：东南西北分别联想新机、喜气、收获与蓄势。
- 数字：1 至 9 分别组合成开路、聚福与顺势的吉利解释。
- 颜色：朱红、墨绿、江蓝、鎏金分别联想行动、安稳、流动与收获。
- 1089：三位数首尾至少相差 2，倒序相减后再与差的倒序相加，得到 1089。
- 普通聊天：本次不使用任何小术，只根据游客的真实回答给祝福。

数字规律要说明是数学趣味，颜色、方位和景物要说明是民俗娱乐联想。所有解释都只能是吉利祝福，不代表真实吉凶，也不能盖过游客明确说出的商品偏好。

## 十、开摊检查

- [ ] 电脑充满电并连接电源。
- [ ] 浏览器已允许麦克风。
- [ ] 真实模型接口测试通过。
- [ ] 语音输入能识别普通话。
- [ ] 语音输出音量适中。
- [ ] 断网演示模式测试通过。
- [ ] 快捷回答可完成完整流程。
- [ ] 算命先生主视觉和香囊、福袋、头巾商品图加载正常。
- [ ] 小术只在对话中自然出现，没有独立入口或强制参与。
- [ ] 推荐只显示一个商品。
- [ ] 工作人员知道敏感话题停止销售。
