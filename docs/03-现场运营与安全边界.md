# 现场运营与安全边界

## 一、对外定位

统一称为：

- 民俗娱乐体验。
- 数学小魔术。
- 水浒人物祝福推荐。
- 今日福物点选。

不要称为：

- 真正预测命运。
- 风水大师开运。
- 医疗或健康诊断。
- 保证发财、升学或治病的服务。

界面右上角已经常驻显示“民俗娱乐体验”。

## 二、工作人员开场

> “可以和四福先生说几句话，玩一个数学小魔术或民俗娱乐签。最后它只为您推荐一件水浒福物，不是算真实命运。”

## 三、游客拒绝购买

当游客说“不买”“只玩玩”“没兴趣”时：

1. 立即停止商品引导。
2. 给一句普通祝福。
3. 点击“重新开始”，清空上一位游客会话。

不要用免费体验制造亏欠感。

## 四、儿童使用

- 儿童由监护人陪同。
- 不询问学校、住址、电话号码和真实姓名。
- 不让儿童为了抽隐藏款连续购买福袋。
- 不引导儿童讨论家庭经济、疾病或父母关系。
- 福袋内小零件按实际年龄提示管理。

## 五、敏感话题处理

### 疾病和健康

只说“祝愿平安、身心舒展”。不能诊断、治疗或推荐香囊代替医疗。

### 财务困难

不推荐财运产品，不暗示消费能改变经济状况。可以结束为普通祝福。

### 考试与升学

可以说“祝学习顺利、从容发挥”，不能说“必过”“保上岸”。

### 丧失、灾害与严重情绪危机

停止娱乐式算命和销售。建议游客寻求家人、现场工作人员或专业帮助。

## 六、隐私

- 默认不保存游客音频和聊天内容。
- 浏览器刷新或按“重新开始”后清空当前前端会话。
- 服务端没有数据库。
- 不进行人脸识别。
- 不要求登录、手机号或扫码注册。
- 若未来增加数据统计，只记录匿名数量，不记录原始语音和完整对话。

## 七、密钥安全

- API key 只放在 `.env`。
- `.env` 已加入 `.gitignore`。
- 不把 key 写进 `src/`、HTML 或二维码。
- 不在游客可见屏幕打开 `.env`。
- 活动结束后如使用临时 key，建议立即轮换或撤销。

## 八、网络与排队

- 每组语音互动控制在 2—4 分钟。
- 排队超过三组时，优先让游客使用快捷回答。
- 模型延迟过高时切换 `MOCK_MODE=true`。
- 每次服务完成后点击“重新开始”。
- 工作人员负责拿出推荐商品，不让游客翻动全部库存寻找。

## 九、三种玩法说明

### 1089 好汉点将术

数学恒等规律。三位数首尾至少相差 2，倒序相减后再与差的倒序相加，得到 1089。

### 6174 聚义回环术

Kaprekar 常数规律。四位数字不完全相同，按降序与升序排列后相减，反复操作最多七步到 6174。

### 四方时运娱乐签

不是数学规律，只是方位联想：

- 东方：启程，张顺。
- 南方：添喜，李逵。
- 西方：放下，鲁智深。
- 北方：守安，安道全。

必须明确告诉游客它是民俗娱乐，不代表真实方位吉凶。

- [ ] 快捷回答可完成完整流程。
- [ ] 四张人物图加载正常。
- [ ] 三种小术均能打开和关闭。
- [ ] 推荐只显示一个商品。
- [ ] 工作人员知道敏感话题停止销售。
