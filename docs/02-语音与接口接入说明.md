# 语音与接口接入说明

## 一、系统结构

```text
浏览器麦克风
  -> 浏览器听写或 STT API
  -> 本地 Express 服务
  -> 大语言模型
  -> 结构化商品推荐
  -> 浏览器朗读或 TTS API
  -> 扬声器
```

浏览器不直接访问模型供应商，所有带 API key 的请求由本地 Express 服务发出。

## 二、为什么分成三个接口

大语言模型不等于语音模型。完整语音聊天至少需要：

1. STT：把游客声音变成文字。
2. LLM：理解游客并生成下一句话或商品结论。
3. TTS：把先生回复变成声音。

三个服务可以来自同一供应商，也可以各用不同供应商。

## 三、延迟目标

适合现场互动的目标：

| 环节 | 建议时长 |
|---|---:|
| 录音结束到识别完成 | 0.5—2 秒 |
| 文本模型回复 | 0.8—3 秒 |
| 语音开始播放 | 0.5—2 秒 |
| 用户体感总等待 | 尽量小于 5 秒 |

如果单轮超过 8 秒，游客会明显觉得卡顿。应优先使用较快模型、限制回复长度，并保持本地规则降级。

## 四、音频建议

- 浏览器录音默认使用 `audio/webm`。
- 单次语音建议 3—15 秒。
- 前端允许游客点击开始、再次点击结束。
- 服务端限制单个文件 12MB。
- STT 固定提示语言为中文。
- TTS 回复限制在约 500 字以内，实际提示词要求 70 个汉字左右。

## 五、供应商适配点

所有供应商差异集中在 `server/providers.mjs`：

- `callChatModel()`：文本模型。
- `transcribeAudio()`：语音识别。
- `synthesizeSpeech()`：语音合成。

如果供应商使用 `X-API-Key` 而不是 Bearer token，只需要修改这三个函数的请求头。

如果文本响应不是 `choices[0].message.content`，修改 `callChatModel()` 的解析部分。

如果 TTS 返回 JSON 内的音频 URL，而不是直接返回二进制，修改 `synthesizeSpeech()` 的响应解析。

## 六、现场语音质量

景区噪声会比室内明显。建议：

1. 麦克风离游客 20—50cm。
2. 使用指向性麦克风。
3. 一次只让一人讲话。
4. 界面保留快捷回答按钮，识别失败时游客仍能完成流程。
5. 工作人员不要和游客同时说话。
6. 音箱面向当前游客，不要把声音开到整个摊位区域。

## 七、故障降级

### 文本模型失败

服务端自动调用 `server/mock-agent.mjs`，根据本地关键词继续推荐。

### STT API 失败

把 `.env` 改为：

```dotenv
STT_PROVIDER=browser
```

或直接让游客点击快捷回答、使用键盘输入。

火山语音合成 2.0 的 API Key 不能默认用于录音文件识别 1.0。出现 `[resource_id=vc.async.default] requested resource not granted` 时，必须保持浏览器听写，或者另行填写录音文件识别页面提供的 `VOLCENGINE_ASR_API_KEY` / `VOLCENGINE_ACCESS_TOKEN`。

### TTS API 失败

把 `.env` 改为：

```dotenv
TTS_PROVIDER=browser
```

系统会调用 Windows/浏览器中文声音。

### 完全断网

```dotenv
MOCK_MODE=true
STT_PROVIDER=browser
TTS_PROVIDER=browser
```

注意：浏览器语音识别本身可能依赖网络。完全断网时，快捷按钮与键盘仍然可用，系统朗读是否离线取决于 Windows 已安装的中文语音。

## 八、真实接口验收用例

接入后至少测试：

1. “我带孩子来，想亲手做一个开心的。”
   - 预期：李逵·快乐加倍 DIY 香囊。
2. “给父母带一个，全家平安，赶时间。”
   - 预期：安道全·安到全家成品香囊。
3. “我旅行想顺利，喜欢兵器和开袋惊喜。”
   - 预期：张顺·顺风顺水随机福袋。
4. “最近工作压力大，想直接带走。”
   - 预期：鲁智深·拔掉烦恼成品香囊。
5. “我不想买，只想玩一玩。”
   - 预期：友好祝福结束，不继续推销。
